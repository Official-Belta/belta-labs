1. Abstract

BELTA Labs는 Uniswap V3/V4 유동성 공급자(LP)가 겪는 비영구적 손실(Impermanent Loss, IL)을 자동 헤징하는 DeFi 프로토콜입니다.

LP는 프리미엄(수수료 수익의 12%)을 납부하고, BELTA의 보험 풀(Underwriter Pool)로부터 IL의 최대 45%를 보상받습니다. Underwriter는 프리미엄 수익과 함께 3가지 추가 수익원을 통해 안정적 APY를 확보합니다.

LAYER 1

Dynamic Fee Hook

변동성 기반 스왑 수수료 동적 조정. 고변동성 시 수수료 수익 2~2.5배 증가.

LAYER 2

Aave Yield Stacking

유휴 풀 자금의 70%를 Aave V3에 예치. 연 4~8% 기본 수익.

LAYER 3

Perps Delta-Hedge

무기한 선물 숏으로 IL 익스포저의 50~70% 상쇄. 블랙스완 손실 대폭 감소.

2. The Problem: Impermanent Loss

Uniswap V3의 집중 유동성(concentrated liquidity)은 자본 효율을 크게 높이지만, IL 리스크도 비례하여 증가합니다.

V3 집중 유동성의 IL 증폭

IL_V3 = IL_V2 × 1 / (1 - √(P_lower / P_upper))

예: ETH/USDC ±10% 범위 설정 시, IL은 V2 대비 약 5배 증폭됩니다. ETH -30% 하락 시 V2 IL 약 2%이지만, 좁은 범위 V3에서는 10~15%까지 확대됩니다.

현재 LP의 딜레마

  • 넓은 범위: IL은 낮지만 자본 효율 저하 → 수수료 수익 미미
  • 좁은 범위: 수수료 수익 극대화, 그러나 IL 리스크 극단적
  • 헤징 부재: 개인 LP는 옵션/선물 헤징을 직접 실행할 수 없음

BELTA는 이 딜레마를 해결합니다: LP가 좁은 범위의 고수익 전략을 안전하게 실행할 수 있도록 IL 리스크를 프로토콜이 흡수합니다.

3. Solution: Insurance Pool + 3-Layer Hedging

LP (Uniswap V4 Position) │ ├─ Premium (fee income × 12%) ──────────> BELTA Protocol │ │ │ ┌────────┴────────┐ │ IL Coverage Underwriter Pool │ (max 45%) │ │ │ ┌──────┼──────┐ └─ IL Reimbursement <────────────┘ Layer1 Layer2 Layer3 DynFee Aave Perps +3~8% +4~8% -50~70% APY APY IL offset

3.0 Base: IL Insurance Pool

LP (보험 가입자)

  • Uniswap V4 LP 포지션을 BELTA Hook에 등록
  • 7일 에포크마다 수수료 수익의 12%를 프리미엄으로 납부
  • 에포크 종료 시 순 IL(수수료 수익 차감 후)의 45%를 보상받음

Underwriter Pool (보험 인수자)

  • USDC를 예치하여 IL 보상 지급 자금 제공
  • 프리미엄 수익 + 3-Layer 추가 수익으로 APY 확보
  • 일일 지급 상한 5%로 풀 고갈 방지

Treasury (1차 손실 흡수)

  • Senior Pool의 20% 규모 유지
  • 블랙스완 이벤트 시 first-loss 흡수 (30%)
  • 자기치유(Self-Healing): 버퍼 부족 시 프리미엄 수익 리다이렉트

3.1 Layer 1: Dynamic Fee Hook

LAYER 1

Uniswap V4의 beforeSwap Hook을 활용하여, 실시간 변동성에 따라 스왑 수수료를 동적으로 조정합니다.

변동성-수수료 곡선 (Aave-style Kink)

변동성 구간스왑 수수료수익 배수
Low (≤30%)0.05%0.7x
Medium (30~50%)0.05% ~ 0.30%1.0x
Kink (50%)0.30%1.0x (기준)
High (50~80%)0.30% ~ 1.00%1.5~2.5x

EWMA 변동성 추적

EWMA_new = 0.3 × Vol_realized + 0.7 × EWMA_old

지수가중이동평균(EWMA)으로 조작 저항성 확보. Keeper가 주기적으로 업데이트.

왜 Dynamic Fee인가?

  • LVR(Loss-Versus-Rebalancing) 환수: 고변동성 시 차익거래자가 LP에서 추출하는 가치를 높은 수수료로 회수
  • IL과 수익의 자연 상관관계: IL이 커지는 고변동성 구간에서 수수료 수익도 증가하여 자연 헤지
  • 예상 효과: 연간 LP 수수료 APY +3~8% 추가

3.2 Layer 2: Aave Yield Stacking

LAYER 2

Underwriter Pool과 Treasury의 유휴 자금을 Aave V3에 예치하여 기본 수익을 창출합니다.

자금 배분

구분Aave 예치 비율유동성 유지예상 APR
Pool 유휴 자금70%30% (IL 클레임 대비)4~8%
Treasury 버퍼70%30% (긴급 지급용)4~8%

수익 흐름

USDC (idle) │ ├─ 70% ──> Aave V3 supply() │ │ │ aUSDC balance grows │ │ │ harvestYield() ──> 수익을 buffer로 회수 │ └─ 30% ──> 유동성 유지 (IL claim 즉시 지급)
  • 최소 예치 금액: 100 USDC (dust deposit 방지)
  • 긴급 시 withdrawFromAave()로 즉시 회수
  • 수익은 Treasury 버퍼에 누적 → 자기치유 가속

3.3 Layer 3: Perps Delta-Hedging

LAYER 3

무기한 선물(Perpetual Futures) 숏 포지션으로 IL 익스포저를 직접 상쇄합니다. 전통 금융의 델타 헤징 전략을 DeFi에 적용한 것입니다.

헤지 전략

파라미터값설명
Target Hedge Ratio50%헤지 LP 총 가치의 50% 숏
Hedge Effectiveness60%IL 오프셋 비율 (불완전 헤지)
Max Leverage3x최대 레버리지 제한
Rebalance Threshold10%편차 초과 시 리밸런싱
Min Rebalance Interval4시간과도한 리밸런싱 방지
Funding Cost~3% APR펀딩비 연간 비용

PnL 흐름

ETH 가격 하락 시: IL 클레임 증가 ──> 풀 부담 ↑ 숏 포지션 이익 ──> 풀 수익 ↑ (상쇄) ETH 가격 상승 시: IL 클레임 감소 ──> 풀 부담 ↓ 숏 포지션 손실 ──> 풀 비용 ↑ (상쇄) → 양방향 대칭적 헤지

지원 프로토콜

IPerpsAdapter 인터페이스로 dYdX, GMX, Hyperliquid 등 주요 무기한 선물 프로토콜과 통합. 체인별 최적 프로토콜 선택 (Arbitrum: GMX, Cross-chain: Hyperliquid).

6. IL Calculation

V2 기본 IL 공식

IL_V2 = 2√r / (1 + r) - 1     (r = P_end / P_start)

V3 집중 유동성 보정

IL_V3 = IL_V2 × 1 / (1 - √(P_lower / P_upper))

P_lower와 P_upper는 LP의 가격 범위. 범위가 좁을수록 보정계수가 커져 IL이 증폭됩니다.

순 IL (Net IL)

Net_IL = max(0, IL_V3 - Fee_Earned)

에포크 동안 LP가 벌어들인 수수료를 차감한 순 IL만 보상 대상입니다. Coverage는 이 순 IL의 최대 45%입니다.

7. Dynamic Premium Pricing

프리미엄율은 풀 이용률(Utilization)에 따라 Aave 스타일 Kink 곡선으로 동적 조정됩니다.

U = 현재 헤지 TVL / Pool 총 TVL
이용률 구간프리미엄율효과
U < 80%base rate (12%)정상 운영
U ≥ 80%12% + slope × (U-80%)/20%최대 2~3배 급등

이용률 80% 이상에서 프리미엄이 급격히 상승하여 과도한 헤지 수요를 자연스럽게 억제합니다. 이는 Aave V3의 이자율 모델과 동일한 검증된 메커니즘입니다.

백테스트 해석 주의사항

V4 수치는 모든 레이어가 완벽하게 작동하는 이상적 시나리오입니다. 실제 운영에서는 다음 요인이 수익을 감소시킬 수 있습니다:

  • Perps 슬리피지, 청산 리스크, 펀딩 비용 변동
  • Aave 이자율 변동 (현재 4~8%이지만 시장 상황에 따라 변동)
  • Dynamic Fee로 인한 LP 이탈 가능성 (과도한 수수료 시)
  • 가스비, 키퍼 비용, 오라클 지연

보수적 추정: 실제 V4 APY는 백테스트의 40~60% 수준으로 예상합니다.
